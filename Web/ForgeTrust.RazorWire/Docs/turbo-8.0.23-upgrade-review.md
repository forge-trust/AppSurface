# Turbo 8.0.23 Upgrade Review

This review records the compatibility analysis for the package-owned Turbo upgrade in AppSurface issue [#656](https://github.com/forge-trust/AppSurface/issues/656). It covers npm `@hotwired/turbo` 8.0.12 source commit [`ea54ae5a`](https://github.com/hotwired/turbo/commit/ea54ae5ad4b6b28cb62ccd62951352641ed08293) through GitHub tag [`v8.0.23`](https://github.com/hotwired/turbo/releases/tag/v8.0.23).

The review includes production behavior, deprecations, and distributed-payload changes that can affect RazorWire. Upstream-only CI, test-framework, documentation, and development-dependency changes are excluded unless they changed the published UMD payload. The npm package is authoritative for shipped bytes; the GitHub comparison is authoritative for behavioral review.

## Result

The matrix requires three focused Chromium regressions: hash-only navigation, resolved parent-frame targeting/header propagation, and inline/before-after stream execution with duplicate sibling removal. All other included changes are already covered by the public workflow matrix, intentionally outside RazorWire's contract, or accepted upstream implementation behavior.

This upgrade does not broaden compatibility guarantees for `HostManaged` or custom Turbo versions. It changes only the exact package-owned default from 8.0.12 to 8.0.23.
